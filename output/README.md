# Output Directory

This directory contains generated agentic documentation and artifacts from the skill library.

## Directory Structure

```
output/
├── agentic/          # Generated agentic documentation
│   └── {repo-name}/
│       ├── AGENTS.md
│       └── agentic/
│           ├── design-docs/
│           ├── domain/
│           └── QUALITY_SCORE.md
├── graphs/           # Generated knowledge graphs
│   └── {repo-name}/
│       ├── graph.json           # NetworkX node-link format
│       └── graphify-out/        # Graphify HTML output (optional)
│           ├── index.html
│           ├── graph.json
│           └── GRAPH_REPORT.md
└── databases/        # SQLite databases
    └── {repo-name}.db           # GitHub/JIRA data

```

## Generated by Skills

### `/create` Skill Output

When you run `/create` in Claude Code, the output is generated in:

**Location**: `output/agentic/{repository-name}/`

**Files Created**:
- `AGENTS.md` - Entry point (≤150 lines)
- `agentic/design-docs/components/` - Component docs (≤100 lines each)
- `agentic/domain/concepts/` - Concept docs (≤75 lines each)
- `agentic/QUALITY_SCORE.md` - Quality metrics

### Knowledge Graph Output

**Location**: `output/graphs/{repository-name}/`

**Files Created**:
- `graph.json` - NetworkX graph (node-link format)
- `graphify-out/` - Graphify HTML visualization (if graphify integration used)

### Database Output

**Location**: `output/databases/{repository-name}.db`

**Contains**:
- GitHub PRs and issues
- JIRA issues and correlations
- Repository metadata

## Usage

### View Generated Documentation

```bash
# View entry point
cat output/agentic/{repo-name}/AGENTS.md

# View component documentation
ls output/agentic/{repo-name}/agentic/design-docs/components/

# View quality score
cat output/agentic/{repo-name}/agentic/QUALITY_SCORE.md
```

### View Knowledge Graph

```bash
# View graph in browser (if graphify used)
open output/graphs/{repo-name}/graphify-out/index.html

# Query graph programmatically
python -c "
import json
with open('output/graphs/{repo-name}/graph.json') as f:
    graph = json.load(f)
print(f'Nodes: {len(graph[\"nodes\"])}')
print(f'Edges: {len(graph[\"links\"])}')
"
```

### Query Database

```bash
# View database statistics
sqlite3 output/databases/{repo-name}.db "
SELECT 
  (SELECT COUNT(*) FROM github_pull_requests) as prs,
  (SELECT COUNT(*) FROM github_issues) as issues,
  (SELECT COUNT(*) FROM jira_issues) as jira_issues;
"
```

## Cleanup

To remove all generated outputs:

```bash
# Remove all outputs
rm -rf output/agentic/* output/graphs/* output/databases/*

# Remove specific repository
rm -rf output/agentic/{repo-name}
rm -rf output/graphs/{repo-name}
rm output/databases/{repo-name}.db
```

## .gitignore

Generated outputs are git-ignored by default to keep the repository clean. If you want to commit specific outputs, use:

```bash
git add -f output/agentic/{repo-name}/AGENTS.md
```

## Best Practices

1. **Use descriptive repository names**: Outputs are organized by repository name
2. **Review before cleanup**: Generated documentation may contain valuable insights
3. **Backup databases**: SQLite databases contain historical GitHub/JIRA data
4. **Version control**: Consider committing quality documentation to share with team

## Troubleshooting

### "Output directory not found"

Create the output directory structure:
```bash
make output
# or
mkdir -p output/{agentic,graphs,databases}
```

### "Permission denied"

Ensure write permissions:
```bash
chmod -R u+w output/
```

### "Broken links in graphify output"

Use the internal knowledge graph instead:
```bash
# The /create skill generates output/graphs/{repo}/graph.json
# This has no HTML link issues
```

Graphify HTML links may break if documentation files are moved. Use the NetworkX JSON graph for reliable programmatic access.
